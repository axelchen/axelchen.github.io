What’s Symbol (符號)？

剛開始學Ruby 常會看到這些 :name, :age ……等等，只知道看到這些前面有:id 的符號，
它叫symbol ，其他仍然一頭霧水，直到某天突然頓悟了ＸＤ
到底什麼是Synbol?
首先，先去Ruby API 查詢一下定義

Symbol objects represent names and some strings inside the Ruby interpreter. They are generated using the :name and :“string” literals syntax, and by the various to_sym methods. The same Symbol object will be created for a given name or string for the duration of a program’s execution, regardless of the context or meaning of that name.

翻譯蒟蒻：

Symbol 物件表示 Ruby直譯器中的「名稱」及某些「字串」。
Symbol 可以用：name和：“string” 語法以及各種 to_sym 方法生成。無論內容或含義為何，都會在程序執行的期間為該名稱或字串創建相同的 Symbol 物件。
看到這邊還是一頭霧水？沒關係一開始我也是！！！
在Ruby的世界裡任何東西都可以是物件（Object），在這邊可以先把符號（Symbol）想成是一個「有名字的物件」，這樣或許會比較好理解，我們常會看到 :name :age這樣的用法，而這種用法我們就稱為 Symbol，而它所指向的地方，其實是指電腦裡記憶體位置。

Symbol跟String有什麼不同？

"String的內容可以改變，但Symbol不行！"
Symbol 是不可變動的( immutable )， String 是可以變動的( mutable )。

這邊我們可以利用以下例子做說明

String

5.times do
puts"hahaha".object_id
end

這邊會得到

70118281331520
70118281331080
70118273359620
70118273359300
70118273358360

Object_id方法會取得Ruby中唯一的數字編號，從上面例子可以發現Ruby在每次產生新的字串時，都會產生新的Object_id，代表這產生的字串其實都是不同的Object。

Symbol

5.times do
puts :hahaha.object_id
end

這邊會得到

1056028
1056028
1056028
1056028
1056028

有發現嗎？只要都是一樣的Symbol，就會有相同的Object_id，表示這些Symbol都是從記憶體一樣的地方拿取資料，而不是產生新的一份資料，這樣的好處是可以節省記憶體的空間。

Symbol & String 轉換方法

使用 to_sym 可把字串轉成 symbol
=> “cat”.to_sym
=> :cat

使用 to_s 方法可以把 symbol 轉成字串
=> :cat.to_s
=> “cat”

什麼時候該用Symbol & String ?

因為Symbol不可變( immutable )特性，以及從記憶體查找資料的速度比String還要快，適合用來當Hash的Key來找尋資料。

範例

data = { name: “小明” , age: 22 , degree: “大學” }
=>{ :name=>“小明”, :age=> 22 , :degree=> “大學” }

而如果單純只是想把資料印出來，那String就很適合，因為Symbol被輸出時，要先轉換成字串，這樣效能上字串上就佔有優勢！

範例

puts “Hello, World”
